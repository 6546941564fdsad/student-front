## 🔧 待优化项（建议逐步实施）

### 1. 安全优化 🔒

#### 1.1 密码加密
- **当前问题**: 密码明文存储
- **建议方案**: 使用 BCrypt 加密
- **实施步骤**:
  ```java
  // 添加依赖
  <dependency>
      <groupId>org.springframework.security</groupId>
      <artifactId>spring-security-crypto</artifactId>
  </dependency>
  
  // 使用 BCryptPasswordEncoder
  BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
  String encodedPassword = encoder.encode(rawPassword);
  boolean matches = encoder.matches(rawPassword, encodedPassword);
  ```

#### 1.2 JWT 拦截器
- **当前问题**: 每个接口手动验证 Token
- **建议方案**: 创建全局拦截器
- **实施步骤**:
  ```java
  @Component
  public class JwtInterceptor implements HandlerInterceptor {
      @Override
      public boolean preHandle(HttpServletRequest request, 
                               HttpServletResponse response, 
                               Object handler) {
          // 验证 Token
          // 设置用户信息到请求上下文
      }
  }
  ```

#### 1.3 输入校验增强
- **当前问题**: 部分接口缺少参数校验
- **建议方案**: 使用 `@Valid` 注解 + 自定义校验注解
- **示例**:
  ```java
  @PostMapping
  public ApiResponse<?> addUser(@Valid @RequestBody UserDTO user) {
      // 自动校验
  }
  ```

### 2. 代码规范优化 📝

#### 2.1 统一日志规范
- **建议**: 使用 SLF4J + Logback
- **日志级别**:
  - `ERROR`: 系统错误
  - `WARN`: 潜在问题
  - `INFO`: 重要业务流程
  - `DEBUG`: 调试信息

#### 2.2 常量提取
- **建议**: 创建常量类，避免硬编码
  ```java
  public class UserStatus {
      public static final String PENDING = "待审核";
      public static final String NORMAL = "正常";
      public static final String DISABLED = "禁用";
  }
  ```

#### 2.3 DTO/VO 分离
- **当前问题**: 直接使用 Entity 返回给前端
- **建议**: 创建 DTO（数据传输对象）和 VO（视图对象）
  ```java
  // UserDTO.java - 用于接收请求
  public class UserDTO {
      private String username;
      private String password;
      // 省略敏感字段
  }
  
  // UserVO.java - 用于返回响应
  public class UserVO {
      private Long id;
      private String username;
      private String name;
      // 不包含密码
  }
  ```

### 3. 前端优化 🎨

#### 3.1 统一错误处理
- **建议**: 创建全局错误处理拦截器
  ```javascript
  // src/utils/errorHandler.js
  export const errorHandler = {
    handle401() { /* Token过期处理 */ },
    handle403() { /* 权限不足处理 */ },
    handle500() { /* 服务器错误处理 */ }
  }
  ```

#### 3.2 加载状态优化
- **建议**: 创建全局 Loading 组件
- **使用**:
  ```vue
  <template>
    <a-spin :spinning="loading">
      <!-- 内容 -->
    </a-spin>
  </template>
  ```

#### 3.3 API 封装优化
- **建议**: 统一 API 响应处理
  ```javascript
  // src/api/base.js
  export const baseApi = {
    request(method, url, data) {
      return axios({ method, url, data })
        .then(res => {
          if (res.data.success) {
            return res.data.data
          } else {
            throw new Error(res.data.message)
          }
        })
    }
  }
  ```

### 4. 性能优化 ⚡

#### 4.1 数据库查询优化
- **建议**:

  - 使用分页查询（避免全表扫描）
  - 延迟加载（@OneToMany(fetch = FetchType.LAZY)）

#### 4.2 前端缓存策略
- **建议**:
  - 使用 localStorage 缓存用户信息
  - 使用 sessionStorage 缓存临时数据
  - 接口请求防抖/节流

#### 4.3 懒加载
- **建议**:
  - 路由懒加载（Vue Router）
  - 组件按需加载
  - 图片懒加载
