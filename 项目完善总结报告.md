  - 课程总数
  - 最高分
  - 最低分
  - 平均分

**技术亮点**:
```javascript
// 自动计算GPA
computed: {
  gpa() {
    if (this.grades.length === 0) return 0;
    const validGrades = this.grades.filter(g => g.gradePoint);
    const sum = validGrades.reduce((acc, g) => acc + g.gradePoint, 0);
    return sum / validGrades.length;
  }
}
```

## 💡 关键技术决策

### 1. 为什么选择多阶段Docker构建？

**优点**:
- 减小最终镜像体积（从~800MB降至~200MB）
- 提高安全性（运行阶段不包含构建工具）
- 加快部署速度

### 2. 为什么统一使用 `request` 而非 `axios`？

**原因**:
- `request` 已配置JWT拦截器
- 统一的错误处理
- 自动添加Token
- 代码一致性

### 3. 为什么学生组件要传递 `user` prop？

**目的**:
- 实现数据隔离
- 避免全局状态污染
- 便于单元测试
- 支持多租户扩展
