# 500 错误修复报告 - 路径重复问题

## 🐛 问题描述

控制台出现多个 500 Internal Server Error：

```
GET http://localhost:8080/api/statistics/overview 500
GET http://localhost:8080/api/api/teacher-evaluations?page=0&size=10 500
GET http://localhost:8080/api/api/teacher-evaluations/statistics 500
```

**注意**: 某些请求路径中出现了 **重复的 `/api`**，例如 `/api/api/teacher-evaluations`。

---

## 🔍 根本原因分析

### 问题架构

系统存在三层 `/api` 路径叠加：

1. **后端 context-path**（application.properties）：
   ```properties
   server.servlet.context-path=/api
   ```
   所有请求自动加上 `/api` 前缀

2. **后端 Controller @RequestMapping**：
   ```java
   @RequestMapping("/api/statistics")
   @RequestMapping("/api/teacher-evaluations")
   // ... 其他 23 个 Controller
   ```

3. **前端 baseURL**（axios.js）：
   ```javascript
   baseURL: 'http://localhost:8080/api'
   ```

### 路径拼接结果

```
实际请求路径 = baseURL + API路径
             = /api + /api/teacher-evaluations
             = /api/api/teacher-evaluations ❌

到达后端路径 = context-path + RequestMapping
            = /api + /api/teacher-evaluations
            = /api/api/teacher-evaluations ❌
```

**结果**: 前端请求 `/api/teacher-evaluations`，实际到达后端 `/api/api/teacher-evaluations`，导致 404 或 500 错误。

---

## ✅ 修复方案

### 策略：保留 context-path，清理冗余 `/api` 前缀

**原则**:
- ✅ `server.servlet.context-path=/api` 保持不变（统一入口）
- ✅ 所有 Controller 去掉 `/api` 前缀
- ✅ 前端 baseURL 保留 `/api`
- ✅ 前端 API 路径去掉 `/api` 前缀

**修复后路径**:
```
前端请求: /api + /teacher-evaluations = /api/teacher-evaluations ✅
后端接收: /api (context) + /teacher-evaluations (RequestMapping) = /api/teacher-evaluations ✅
```

## 🎯 路径规则统一

### 后端配置（application.properties）
```properties
server.servlet.context-path=/api
```

### 前端配置（axios.js）
```javascript
const api = axios.create({
  baseURL: 'http://localhost:8080/api', // 包含 context-path
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
});
```

### Controller 标准格式
```java
@RestController
@RequestMapping("/resource-name")  // ⚠️ 不要加 /api 前缀
@CrossOrigin(origins = "*")
public class ResourceController {
    // ...
}
```

### 前端 API 标准格式
```javascript
import request from './axios';

export const resourceApi = {
  // ⚠️ 不要加 /api 前缀
  getAll(params = {}) {
    return request.get('/resource-name', { params });
  },
  
  getById(id) {
    return request.get(`/resource-name/${id}`);
  },
  
  create(data) {
    return request.post('/resource-name', data);
  },
  
  update(id, data) {
    return request.put(`/resource-name/${id}`, data);
  },
  
  delete(id) {
    return request.delete(`/resource-name/${id}`);
  }
};
```

## 💡 经验教训

### 避免路径重复的最佳实践

1. **统一入口点**: 使用 `server.servlet.context-path` 统一管理基础路径
2. **Controller 简洁**: RequestMapping 只写资源名，不加基础路径前缀
3. **前端配置化**: baseURL 包含 context-path，API 路径只写资源名
4. **代码审查**: 新增接口时检查路径是否重复

### 常见错误模式

```java
// ❌ 错误：三层 /api 叠加
// application.properties: server.servlet.context-path=/api
@RequestMapping("/api/users")  // Controller 又加了 /api
// 前端: axios.get('/api/users')  // 前端再加 /api
// 结果: /api/api/api/users ❌

// ✅ 正确：只在 context-path 配置一次
// application.properties: server.servlet.context-path=/api
@RequestMapping("/users")  // Controller 不加 /api
// 前端: axios.get('/users')  // 前端不加 /api
// 结果: /api/users ✅
```
