# 组件联动与代码优化指南

## 📋 优化概述

本次优化解决了以下问题:
1. ✅ Dashboard 统计数据不会随其他模块数据变化而自动更新
2. ✅ 各模块之间缺乏通信机制
3. ✅ 代码冗余和重复逻辑

## 🔧 已完成的优化

### 1. 创建全局事件总线 (EventBus)

**文件**: `src/utils/eventBus.js`

```javascript
import eventBus from '../utils/eventBus';

// 在任何组件中触发事件
eventBus.emit('data-changed', { module: 'grade', action: 'add' });

// 在 Dashboard 中监听
eventBus.on('data-changed', this.handleDataChanged);
```

### 2. Dashboard 增强

**文件**: `src/components/Dashboard.vue`

- ✅ 添加了 EventBus 监听
- ✅ 实现 `handleDataChanged` 方法
- ✅ 组件销毁时自动清理事件监听(防止内存泄漏)
- ✅ 智能判断哪些模块变化需要刷新Dashboard

## 📝 需要在各模块中添加的代码

### 通用模式

在每个执行了增删改操作的模块中,添加以下代码:

#### Step 1: 导入 EventBus

```javascript
import eventBus from '../utils/eventBus';
```

#### Step 2: 在操作成功后触发事件

```javascript
// 示例: 成绩录入模块
async handleSubmit() {
  try {
    await gradeApi.batchAdd(this.gradeList);
    this.$message.success('提交成功');
    
    // ✅ 触发事件通知 Dashboard
    eventBus.emit('data-changed', { 
      module: 'grade',  // 模块标识
      action: 'add'     // 操作类型: add/update/delete/audit
    });
    
    this.loadData();
  } catch (error) {
    this.$message.error('提交失败');
  }
}
```

## 💡 最佳实践

### 1. 事件命名规范
- 使用 `data-changed` 作为统一事件名
- 通过 `module` 和 `action` 参数区分不同场景

### 2. 防抖处理
Dashboard 中已实现 500ms 延迟刷新,避免频繁请求

### 3. 内存管理
所有监听事件的组件必须在 `beforeUnmount` 中移除监听:
```javascript
beforeUnmount() {
  eventBus.off('data-changed', this.handleDataChanged);
}
```

### 4. 错误处理
EventBus 内部已包含 try-catch,单个回调出错不会影响其他监听器
