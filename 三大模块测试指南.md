### 测试场景 1.2：添加评教（正常流程）

**操作步骤：**
1. 点击 **"添加评教"** 按钮
2. 在弹出的表单中填写：
   - 评教类型：选择 **"学生评教"**
   - 教师ID：输入 `1`
   - 教师姓名：输入 `"张老师"`
   - 课程ID：输入 `1`
   - 课程名称：输入 `"Java程序设计"`
   - 评教人ID：输入 `1`
   - 评教人姓名：输入 `"李同学"`
   - 评分：拖动滑块到 `4.5`
   - 评价内容：输入 `"老师讲课很清晰，内容丰富，受益匪浅。"`
   - 评价学期：输入 `"2025-2026-1"`
3. 点击 **"提交"** 按钮

**预期结果：**
- ✅ 表单验证通过（无红色错误提示）
- ✅ 显示 loading 状态
- ✅ 弹出成功提示："评教提交成功！"

- ✅ 列表自动刷新，显示新添加的评教记录
- ✅ 统计数据更新

## ⚠️ 常见错误及排查

### 错误 1：CORS 跨域错误

**现象：**
```
Access to XMLHttpRequest at 'http://localhost:8080/api/...' 
from origin 'http://localhost:5173' has been blocked by CORS policy
```

**解决方案：**
1. 检查后端 `CorsConfig.java` 是否存在
2. 重启后端服务
3. 清除浏览器缓存

---

### 错误 2：404 Not Found

**现象：**
API 请求返回 404

**解决方案：**
1. 检查后端服务是否运行
2. 检查 API 路径是否正确
3. 查看后端控制台是否有启动错误

---

### 错误 3：500 Internal Server Error

**现象：**
API 请求返回 500

**解决方案：**
1. 查看后端控制台错误日志
2. 检查数据库连接是否正常
3. 检查实体类验证注解是否正确

---

### 错误 4：表单验证不生效

**现象：**
提交空表单没有错误提示

**解决方案：**
1. 检查组件中是否有 `ref="formRef"`
2. 检查是否有 `:rules="formRules"`
3. 检查 `handleSubmit` 中是否调用 `validate()`
